# DRAFT ROOM

Firebaseを使った、複数端末対応のリアルタイム・ドラフト会議Webアプリです。
ビルド作業は不要で、HTMLファイル一式を配信するだけで動きます。

## できること

- 6桁コードによる部屋作成・参加
- 参加者一覧と指名順のリアルタイム共有
- 候補選手の自由登録、検索、ポジション絞り込み
- 部屋作成時に2026年度NPB支配下選手を「12球団／セ・リーグ／パ・リーグ」から一括追加
- 初期選手を球団順・背番号順で表示（育成選手は対象外）
- MLB全30球団の現役ロースター、NPB12球団の監督・コーチをテンプレートから追加
- 複数テンプレートの組み合わせと重複選手の自動除外
- 独自選手テンプレートの作成・端末保存・再利用
- 順番指名（Firestoreトランザクションで二重指名を防止）
- 全員同時指名
- 同名指名の自動検出と、その場での抽選
- 主催者による重複の目視確認
- PC・スマートフォン対応

## Firebaseの準備

1. [Firebase Console](https://console.firebase.google.com/)でプロジェクトを作成します。
2. 「Authentication」→「始める」→「匿名」を有効にします。
3. 「Firestore Database」→「データベースを作成」を選びます。
4. Firestoreの「ルール」画面へ `firestore.rules` の内容を貼り付け、公開します。
5. 「プロジェクトの設定」→「マイアプリ」からWebアプリを追加します。
6. 表示された `firebaseConfig` の値を `firebase-config.js` へ貼り付けます。

## 起動方法

ES Modulesを使っているため、`index.html` を直接ダブルクリックせず、ローカルサーバーで開きます。

Visual Studio Codeを使う場合は「Live Server」拡張機能で `index.html` を開くのが簡単です。

Firebase Hostingを使う場合：

```sh
npm install -g firebase-tools
firebase login
firebase init hosting
firebase deploy
```

`firebase init hosting` では公開フォルダに `.`（現在のフォルダ）を指定し、SPA設定は「No」を選びます。

## 指名方式

- **順番に指名**：参加順で指名します。選手が確定した瞬間に、次の参加者へ指名権が移ります。
- **全員同時に指名**：各巡目で全員が1名ずつ指名します。同じ選手を選んだ球団がある場合は抽選画面が全端末に表示されます。
- **主催者が目視判定**：現在の指名が右側へ並びます。表記揺れなどで同一選手だと判断した指名を2件以上チェックしてから締め切ると、その組み合わせで抽選します。完全に同じ名前はチェックなしでも競合として扱われます。

## 注意

この構成は友人・小規模グループ向けです。一般公開して大人数で利用する場合は、Cloud Functionsで抽選と指名確定をサーバー側へ移し、Firestoreルールもさらに厳密にしてください。

初期選手データは、NPB公式サイトの2026年度選手一覧（2026年6月21日現在）の「支配下選手」を基準にしています。移籍・新規登録・登録抹消などの変更は自動では反映されないため、必要に応じて候補選手の編集・削除機能で調整してください。

MLBテンプレートはMLB公式Stats APIの2026年6月22日時点のActive Roster、監督・コーチテンプレートはNPB公式の2026年度監督・コーチ一覧を基準にしています。独自テンプレートはブラウザのローカルストレージへ保存されるため、同じ端末・ブラウザで再利用できます。
